# Official Guidance (Astro + Starlight)

Primary sources used for this skill.

## Astro

- Environment variables:
  - https://docs.astro.build/en/guides/environment-variables/
  - Key points:
    - Only `PUBLIC_` variables are exposed to client-side code.
    - `.env` files are not available in `astro.config.*`.
    - Astro supports typed env schemas via `astro:env`.
- Content collections:
  - https://docs.astro.build/en/guides/content-collections/
  - Key points:
    - Content collections are Astro’s recommended way to manage and type-check structured content.
    - Schemas are powered by `astro/zod`.
- Images:
  - https://docs.astro.build/en/guides/images/
  - Key points:
    - Use `astro:assets` for optimized images.
    - `alt` is required for `Image` and `Picture`.
    - Assets in `public/` are not optimized by Astro.
- Middleware:
  - https://docs.astro.build/en/guides/middleware/
  - https://docs.astro.build/en/reference/modules/astro-middleware/
  - Key points:
    - `defineMiddleware()` is the typed entry point.
    - `context.locals` is the standard request-scoped context surface.
- Rendering modes / SSR:
  - https://docs.astro.build/en/basics/rendering-modes/
  - https://docs.astro.build/en/guides/on-demand-rendering/
  - Key points:
    - Static is default.
    - Use server rendering for request-time behavior.
    - Route-level control via `export const prerender = false/true`.
- Directives:
  - https://docs.astro.build/en/reference/directives-reference/
  - Key points:
    - Client directives (`client:load`, `client:idle`, `client:visible`) control hydration timing.
    - `server:defer` enables server islands.

## Starlight

- Configuration:
  - https://starlight.astro.build/reference/configuration/
  - Key points:
    - `components` option is the official override mechanism.
    - `pagefind` can’t be enabled when Starlight `prerender` is `false`.
- Sidebar:
  - https://starlight.astro.build/guides/sidebar/
  - Key points:
    - Default sidebar can be autogenerated from docs structure.
    - Frontmatter `sidebar` controls label/order/badge/hidden for autogenerated links.
- Frontmatter:
  - https://starlight.astro.build/reference/frontmatter/
  - Key points:
    - Relevant fields: `template`, `pagefind`, `draft`, `sidebar`.
- Site search:
  - https://starlight.astro.build/guides/site-search/
  - Key points:
    - Pagefind search is enabled by default.
    - `pagefind: false` excludes a page.
    - `data-pagefind-ignore` excludes specific sections.
