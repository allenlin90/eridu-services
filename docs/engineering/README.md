## Usage

These documents describe **how the system is built** — not what it does for users (see [docs/domain/](../domain/BUSINESS.md)) and not what features are being built (see [docs/features/](../features/README.md)).

Reference these when making architectural decisions, onboarding to the codebase, or evaluating whether a new convention fits the existing model.
