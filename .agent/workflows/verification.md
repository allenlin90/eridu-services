# Verification Workflow

Run this workflow after making code changes to ensure quality before marking work complete.

> For feature/refactor work, run **Knowledge Sync Workflow** (`.agent/workflows/knowledge-sync.md`) after verification so docs/skills/rules/memory stay current.

## Steps

Determine which app(s) were modified (e.g., `erify_api`, `erify_studios`, `erify_creators`).

// turbo-all

1. **Lint check**
```bash
pnpm --filter <app> lint
```

2. **Type check**
```bash
pnpm --filter <app> typecheck
```

3. **Run tests**
```bash
pnpm --filter <app> test
```

4. **Migration checks (when DB schema/migrations changed)**
   - Confirm new migrations were generated by official tooling, not hand-written from scratch.
   - For `erify_api` (Prisma), run:
```bash
pnpm --filter erify_api prisma migrate status
```
   - If local env requires applying unapplied Prisma migrations:
```bash
pnpm --filter erify_api prisma migrate deploy
```
   - For deterministic local validation on schema iterations, run:
```bash
pnpm --filter erify_api db:local:refresh
```
   - If auth ext-id mappings are needed for cross-app local tests, run:
```bash
pnpm --filter erify_api db:extid:sync
```
   - For `eridu_auth` (better-auth + Drizzle), run:
```bash
pnpm --filter eridu_auth auth:schema
pnpm --filter eridu_auth db:check
```
   - If migration SQL includes manual customization, ensure inline comments explain why tool-generated SQL was insufficient.

5. **Seed + fixture impact gate (when schema OR service contract changed)**
   - Treat seed data as part of runtime contract for local/dev workflows.
   - Review changed models/services and verify whether any seeded reference data must be updated (for example `systemKey`, enum-like status rows, relation UIDs, required fields).
   - If data is stale due to month rollover or cross-app mapping drift, run:
     - `.agent/workflows/seed-refresh-cross-month.md`
   - Re-run deterministic cycle and confirm seed completeness checks do not incorrectly skip required backfills:
```bash
pnpm --filter erify_api db:local:refresh
```
   - If schedule-planning payload/response contracts changed, regenerate and validate manual payload flow:
```bash
pnpm --filter erify_api manual:schedule:generate
```
   - If local API is running, run end-to-end manual schedule smoke:
```bash
pnpm --filter erify_api manual:schedule:all
```
   - Rule: if manual scripts break due to DTO/schema mismatch, fix manual scripts/seed first; avoid changing production services unless product behavior is wrong.

6. **Refactor parity checks (for feature/refactor work)**
   - Confirm loading/empty/data UI states still match expected behavior.
   - Confirm route/search-param behaviors still match expected URL contract.
   - Confirm pagination/date/filter transitions behave correctly after extraction/decomposition.
   - Confirm route-layout DRY parity for touched route sets:
     - parent route owns shared container/guard via `<Outlet />`
     - leaf pages use the route-set shared page wrapper (for example `AdminLayout` or `PageLayout`) instead of repeated manual header markup.

7. **Review results** — If any step fails, fix the errors before marking work complete. Re-run the failing step after fixing.

> **Note**: For changes spanning multiple apps or packages (e.g., `@eridu/api-types` + `erify_api`), run verification for each affected package.
